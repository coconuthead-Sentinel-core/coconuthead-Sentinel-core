## üìã Pull Request Process

1. **Fork & Branch:** Create a feature branch from `main`
2. **Make Changes:** Implement your changes with clear, focused commits
3. **Add Tests:** Include tests for new functionality
4. **Update Docs:** Update README.md if adding features
5. **Test Thoroughly:** Ensure all tests pass
6. **Submit PR:** Open a pull request with a clear description

### PR Guidelines

- **Title:** Use clear, descriptive titles (e.g., "Add ADHD hyperfocus mode support")
- **Description:** Explain what, why, and how
- **Tests:** All tests must pass
- **Code Style:** Follow existing patterns in the codebase
- **Commits:** Use meaningful commit messages

When proposing new features:

1. **Align with Mission:** Does it enhance neurodivergent accessibility?
2. **Clear Use Case:** Describe how it helps users
3. **Maintain Simplicity:** Don't overcomplicate the architecture
4. **Consider Diversity:** How does it work for different cognitive styles?

## üêõ Bug Reports

Good bug reports include:

- **Description:** Clear explanation of the issue
- **Steps to Reproduce:** Exact steps to trigger the bug
- **Expected Behavior:** What should happen
- **Actual Behavior:** What actually happens
- **Environment:** Python version, OS, etc.
- **Code Sample:** Minimal code that reproduces the issue
